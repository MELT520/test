# WEBpack 使用指南

> 将任意网页打包为桌面应用和移动端 APP，免费、全功能、无限制。

---

## 项目简介

WEBpack 是基于 **Tauri 2 + Rust + Vue 3** 开发的网页打包工具，可将任何 URL 或本地 HTML 打包为：

- **桌面端**：Windows (.exe) / macOS (.app) / Linux
- **移动端**：Android (.apk) / iOS (.ipa)

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 🌐 **网页打包** | 输入 URL，一键打包为原生应用 |
| 📱 **移动端支持** | Android APK / iOS IPA 云端构建 |
| 🎨 **自定义图标** | 上传 PNG/JPG/SVG，自动生成各平台图标 |
| ✏️ **自定义 JS** | 注入自定义 JavaScript 代码 |
| 🔧 **窗口配置** | 标题、宽高、居中、全屏、可调大小等 |
| 📂 **本地 HTML** | 打包本地 HTML 文件夹为应用 |
| ☁️ **云端构建** | 通过 GitHub Actions 云端编译移动端 APP |
| 💾 **配置记忆** | 构建历史自动保存（100 条），一键复用 |
| 🔗 **自定义仓库** | 支持自有 GitHub 仓库作为构建后端 |

## 详细使用说明

### 一、桌面端打包

#### 1. 基本配置

| 配置项 | 说明 | 必填 |
|--------|------|------|
| 项目名称 | 生成的应用名称 | ✅ |
| 目标 URL | 要打包的网页地址 | ✅ |
| EXE 名称 | 生成的 .exe 文件名 | ✅ |
| 输出目录 | 打包产物保存位置 | ✅ |
| 窗口标题 | 应用窗口标题栏文字 | 可选 |
| 窗口宽度 | 默认窗口宽度（默认 800） | 可选 |
| 窗口高度 | 默认窗口高度（默认 600） | 可选 |

#### 2. 窗口设置

- **居中显示**：窗口启动时屏幕居中
- **可调整大小**：允许用户拖动边框改变窗口大小
- **最大化**：启动时最大化窗口
- **全屏**：启动时全屏显示

#### 3. 高级功能

- **自定义 JS 注入**：在页面加载完成后执行自定义 JavaScript
- **调试模式**：启用后支持右键打开开发者工具 (F12)
- **本地 HTML 打包**：选择本地 HTML 文件夹，无需服务器即可运行

#### 4. 构建流程

1. 填写项目配置
2. 点击 **"本地构建"** 按钮
3. 等待构建进度条完成
4. 在输出目录找到打包好的 EXE 应用

> **注意**：Windows 构建产物包含 `<名称>.exe`、`config/` 配置目录、`app.ico` 图标文件。发送给他人时请保持目录结构完整。

### 二、移动端云端构建

#### 前置条件

1. **Fork 云端构建仓库**

   将 [MELT520/WEBpack](https://github.com/MELT520/WEBpack) Fork 到你的 GitHub 账号。

2. **创建 GitHub Token**

   访问 [GitHub Settings → Tokens](https://github.com/settings/tokens) 创建 Personal Access Token，勾选以下权限：
   - `repo`（仓库读写）
   - `workflow`（触发 Actions）

#### Android 构建

1. 切换到 **Android** 标签页
2. 配置以下参数：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 项目名称 | 手机上显示的应用名称 | 我的应用 |
| 目标 URL | 应用的网页地址 | https://example.com |
| 包名 | Android 包名（唯一标识） | com.mycompany.myapp |
| 版本号 | 应用版本 | 1.0.0 |

3. **上传应用图标**（可选但推荐）

   点击图标区域，选择 PNG/JPG/SVG 格式的图片（建议 512×512 以上）。

4. 填写 **GitHub Token** 和 **仓库名**（如 `你的用户名/WEBpack`）

5. 点击 **"云端构建"** 按钮

   系统将自动：
   - 上传图标到云端仓库
   - 触发 GitHub Actions 编译
   - 构建完成后自动创建 Release

6. 前往 `https://github.com/你的用户名/WEBpack/actions` 查看构建进度

7. 构建完成后，在 Releases 页面下载 APK 安装

#### iOS 构建

操作步骤与 Android 类似，切换到 **iOS** 标签页即可。

> **注意**：iOS 构建产物为 .ipa 文件，需要通过 AltStore、Sideloadly 等方式侧载安装，或使用 Apple Developer 账号签名。

---

### 三、构建历史

WEBpack 自动保存最近 **100 条**构建记录到本地：

- **查看历史**：主页面底部「构建历史」卡片
- **一键填入**：点击历史记录，自动填充所有配置参数
- **清空历史**：点击「清空历史」按钮清除所有记录

> 注意：图标和自定义 JS 不会被保存到历史中（体积过大），需要重新上传。

---

### 四、配置保存/加载

- **保存配置**：点击「保存配置」将当前设置保存到本地浏览器
- **加载配置**：点击「加载配置」恢复上次保存的设置
- 仅保留一组配置，重复保存会覆盖

## 常见问题

### Q: 构建失败怎么办？
1. 检查网络连接，确保能访问目标 URL
2. 查看 GitHub Actions 日志定位具体错误
3. 确认 Token 权限包含 `repo` 和 `workflow`

### Q: APK 安装后图标不对？
1. 确保在构建前上传了自定义图标（看到"图标已上传"提示）
2. 图标建议使用 512×512 以上的 PNG 格式
3. 部分国产手机启动器可能需要清除桌面缓存

### Q: 应用打开后白屏？
1. 确认目标 URL 可正常访问
2. 检查是否开启了代理/VPN
3. 尝试启用调试模式查看控制台错误

### Q: 如何更新版本号？
在配置界面修改版本号后重新构建。Android 的 `versionCode` 会自动递增，确保覆盖安装正常。

### Q: 本地构建的 EXE 给别人用不了？
确保将整个输出目录打包发送，包括 `config/` 文件夹和 `app.ico` 图标文件。
